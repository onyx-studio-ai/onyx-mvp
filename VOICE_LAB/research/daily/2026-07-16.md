## 重點(可操作)

### 1. 🚨 最大的魚:Higgs Audio v3 主檔標「Apache 2.0 ✅ 可商用」——**實際是禁止商用**

主檔把 Higgs Audio v3 放在**第一梯隊(可上線等級)**,授權寫「Apache 2.0 ✅ / 無 royalty」。查核結果三個欄位**全錯**:

| | 主檔記載 | 實際查證 |
|---|---|---|
| v3 權重授權 | Apache 2.0 | **BOSON HIGGS TTS 3 RESEARCH AND NON-COMMERCIAL LICENSE**(2026-07-08 版,32KB 自訂約) |
| 商用 | ✅ 可 | ❌ **全面禁止**(`commercial` 出現 42 次,v2 曾有的 10 萬 AAU 免費門檻在 v3 直接取消) |
| 拿輸出訓練自家模型 | (未提) | ❌ §IV(b)(i) 禁止「create, train, fine-tune, distill, or **improve any** foundational generative AI model」 |

- **Apache 2.0 只蓋 GitHub 的 code,不蓋權重** —— 教科書級的「徽章 ≠ 權重授權」(通則①),連 Boson 自己 README 都寫明權重走 Research and Non-Commercial License。
- **更正一個直覺**:這不是「v2 是 Apache、v3 偷偷換掉」。查核員讀了 v2 的 commit 歷史 —— v2 從 2025-06-20 首發就是 Community License(改自 Llama 3),**主檔從第一天就抓錯了**(把 GitHub 的 Apache 徽章當成權重授權),v3 只是延續這個錯。v3 確實比 v2 更緊(Community → Non-Commercial)。
- **危險等級高於昨天的 XTTS/F5**:那兩個在**第二梯隊**(標題就寫「可用但有限制」),Higgs 在**第一梯隊**(標題寫「可上線等級」)—— 讀者最信任、選型最先看的地方。權重位置 `bosonai/higgs-tts-3-4b` 是真的、公開的、不用申請,任何人照主檔就會直接上線 → 無授權商用。
- **好消息只有一個**:Higgs 的訓練資料(10M+ 小時)風險 Boson 自己吞了,沒有 F5-TTS/Emilia 那種「訓練資料傳染」,`training data` 只出現在「禁逆向工程萃取」條款。所以它是「公司政策性紅燈」,理論上有商用窗口(contact@boson.ai 談付費授權),不是死路。

**風險定性:潛在地雷,不是正在燒的火。** 已 grep 全 repo 確認 Higgs 只在 `VOICE_PLATFORM_HANDOVER.md` 被列為「⚠️ 未啟動的實驗」,生產線沒用到(現行 GPT-SoVITS + CV3)。所以是「趁沒人照第一梯隊挑它上線前先拆引信」。

### 2. 兩支生產主力複驗綠燈(這是今天最該鬆一口氣的事)

- **CosyVoice 3**(升商用主力)🟢:HF `Fun-CosyVoice3-0.5B-2512` model card 標籤 `apache-2.0`,GitHub LICENSE 與 Apache 官方原文逐字全等(9101 字元)、四關鍵詞全落空。連 RL 版 `llm.rl.pt` 都在同 repo 同授權下。再次確認「for academic purposes only」那句在文件最末 Disclaimer 區、主詞是「上面提供的內容/部分範例取自網路」,是講 demo 樣本不是模型本體(和 C12 判定一致,且這句是三代共用的樣板)。
- **GPT-SoVITS**(現行生產)🟢:HF `lj1995/GPT-SoVITS` 標準未改動 MIT,LICENSE 檔史上只有 1 次初始 commit、從沒加料;內建 vocoder/HuBERT/RoBERTa/SV 全 MIT 或 Apache。「歷史上加過額外條款」的疑慮**不成立**。

**兩支都真的乾淨**,不是「程式碼寬、權重緊」的陷阱 —— 我們運氣好,也是本錢。

### 3. Fish Speech V1.5:主檔紅燈可**降級為黃燈**(有活的商用窗口),兼結一條待辦

- 授權名稱「CC-BY-NC-SA-4.0 / ❌ 非商用」**正確**,但「紅燈=死路」的定性可鬆。fishaudio 公司還在、官方開發者頁**點名 fish-speech**「ship as open weights with a paid commercial license」,自架商用報價 $10k/月起(業務窗口 business@fish.audio)。這和 Coqui 倒閉、F5-TTS 明說不發是不同性質 —— **有窗口=黃(可談),沒窗口=紅**(通則⑤)。
- **順手結掉主檔待辦「fishaudio/fish-speech 看是否改授權成可商用」**:沒有改;沒有 1.6;產品線改名 S1/S2 且**越改越緊**(程式碼從 Apache 收成 Research License)。**結論:等它自己開放是等不到的。**
- 主檔漏記一條硬限制:下載時 gated 要勾「non-commercial use ONLY」的 checkbox,是獨立於 CC 授權的契約承諾,比授權本身更難繞。

### 4. Fish Audio S2:紅燈維持,但「商用只能買它 API」這句**不精確**

- 自架紅燈成立(Research License,gated 要勾非商用)。但主檔「商用只能買 API」會讓我們誤以為沒談判空間 —— 實際上官方 Enterprise 方案列有 **On-Premise Deployment**,自架有得談(細節要寫信問 business@fish.audio,查不到條款全文)。
- **對 Onyx 的紅線(比授權更重要)**:Fish 的 API Terms 明寫上傳的內容「may be used to develop, train, or enhance AI models」,而且 **ZDR(zero data retention)只有 Enterprise 才有**,連 $749/月的 Max 都沒有。**在拿到 Enterprise 書面 ZDR 前,任何客戶/配音員聲音都不該進 Fish 的 API,連免費測試都不行。** 這坐實了記憶 `reference_fish_audio_api.md`,且比記憶更嚴重。

### 5. 兩支綠燈引擎漏了會影響交付的營運註記(建議補進主檔)

- **Chatterbox** 🟢 標準 MIT 無誤,但**每個輸出硬編碼嵌入 PerTh 神經浮水印**(程式碼無開關)。好消息:授權沒禁移除、可 fork 刪掉那行、浮水印只是 1 bit 的「AI 生成」旗標(**不含 Resemble 品牌、不含可回溯 ID**),對客戶交付實務上無影響。真正要想清楚的是**反向** —— 歐盟 AI Act 的 AI 標示義務下,保留浮水印可能反而是合規資產,移除前先確認法規(待查)。
- **GPT-SoVITS** 🟢 權重乾淨,但兩個黃色保留:①底模訓練資料來源官方**從未公開**(無法自證乾淨,但也無 NC 標籤,屬透明度風險)②`g2p_en` 傳遞依賴一個 GPL 套件 `distance`(issue #2776 未解)—— 只在「**打包散布軟體**」時才觸發 GPL,我們 pod 上跑推論當服務不散布程式碼,**現行用法不受影響**,但哪天要把引擎打包給客戶就要換掉。
