# Copy Clarity Criteria

## Purpose
- Ensure every public page answers one core question fast: "What does this platform do for me?"
- Keep copy consistent across `zh-TW`, `en`, and `zh-CN`.

## 4-Line Rule for Above-the-Fold Copy
1. **Service Definition**: What is being sold (voiceover, music production, live strings).
2. **Differentiation**: Why this service is different (AI speed + human direction).
3. **Deliverable**: What the buyer receives (commercial-ready files, license scope, timeline).
4. **CTA**: What to do next (preview, pricing, start order).

## Required Language Style
- Prefer concrete nouns over slogans-only lines.
- Keep one sentence focused on one idea.
- Avoid ambiguous words like "magic", "future", "difference" without context.
- If legal/licensing is mentioned, describe scope and limits in plain language.

## Pricing/Checkout Clarity Rules
- Show the billing unit consistently (e.g., per 1-minute block).
- Avoid hard-coded add-on pricing text when pricing is rule-based.
- Payment CTA should include amount in one natural phrase.
- Tax fields must state whether they are billing-only or licensee-required.

## QA Checklist Before Release
- Can a new user explain the service in 5 seconds after reading first screen?
- Are key terms identical in intent across all three locales?
- Are long sentences still readable on common desktop widths without awkward wraps?
- Are pricing units and license labels consistent from creation flow to checkout?
