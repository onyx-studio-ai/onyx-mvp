# 製作與權利聲明（Provenance）範本

## 目的

- **小案／仲介單**：用「簡版」一頁聲明釐清**誰對素材與成品權利負責**，不必等同 Enterprise 審計包。  
- **大客戶／平台審核**：在簡版上擴充「工具類別、人類審核點、交付索引」，仍保持**不揭露商業機密**為原則。

## A. 簡版（小案／一般 B2B）

**文件標題**：製作與權利聲明（簡版）  
**專案**：{專案名稱}  
**日期**：{YYYY-MM-DD}  
**供應商**：Onyx Studios／{正式公司名稱}

### 1. 素材來源與權利

- 客戶提供之影片、人聲參考、腳本、譯稿、品牌素材，**權利與合法性由客戶負責**；客戶聲明有權委託本專案之改作與多語發行所需之授權。  
- 若由供應商提供庫存音樂／聲線／工具產出之內容，**授權範圍以報價單與合約載明為準**（例如：付費後全平台使用之產品化授權）。

### 2. 製作流程摘要（不揭露專有參數）

- 製作包含：**{例：翻譯／語音合成或對嘴流程／人耳品質檢查／混音與合成}**。  
- **人類審核點**：{例：最終混音前由導演或指定 QC 人員確認可接受度}。

### 3. 交付物

- 交付內容與格式以**報價單「包含」欄**為準（含／不含分軌、5.1、影片檔等）。

### 4. 責任邊界

- 供應商對**交付物之製作品質**依合約負責；對客戶提供素材之**原始權利爭議**不承擔保證責任，除非合約另有約定。

## 誰簽字？

| 情境 | 建議 |
|------|------|
| 小案、簡版 | **供應商**製作或營運主管一人簽即可；客戶未必需簽，視仲介或客戶要求。 |
| 平台／法務要求 | 常見為**雙簽**（客戶採購或製片＋供應商主管）；條款細節回歸 MSA。 |
| 仲介單 | 確認仲介合約是否已涵蓋權利鏈；本聲明仍應與 [QUOTE_TEMPLATE_STANDARD.md](./QUOTE_TEMPLATE_STANDARD.md) 一致，避免責任空洞化。 |
