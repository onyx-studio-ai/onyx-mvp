# Voice Rights Coverage Checklist + Legal Glossary (ZH/EN)

> 用途：給內部法務、BD、PM 在簽約前快速核對「對客承諾」是否完全由人才授權書支撐。  
> Use: Pre-sign review sheet to ensure customer-facing promises are fully backed by talent-side rights.

## A. 覆蓋檢核表 Coverage Checklist

請逐項確認下列條款在最終簽署版本中存在且語義一致：  
Confirm each item exists and is semantically aligned in final execution copy.

- [ ] 永久授權（Perpetual / Irrevocable）  
- [ ] 全球地域（Worldwide / All Territories）  
- [ ] 可再授權（Sublicense to downstream clients/agencies）  
- [ ] 可轉讓（Transfer/Assignment in buyout scenarios）  
- [ ] 成果物與模型權重歸屬甲方（Ownership of derivatives/model weights）  
- [ ] 著作人格權不主張（Moral Rights Covenant Not to Assert）  
- [ ] 客戶端免權利金（Royalty-Free to clients; no residual claims）  
- [ ] Voice ID 宣誓音檔及可驗證用途（Identity/chain verification linkage）  
- [ ] 人才離開不溯及既往（Non-retroactive talent departure）  
- [ ] Archive Status 永久保存但限法律/驗證/合規用途（Purpose-limited permanent archival）  
- [ ] 刪除請求之前提：退回對價＋補償成本（Deletion reimbursement gate）  
- [ ] 3 天不反對視為同意（where legally permissible）  
- [ ] 電子通知送達規則（Email service deemed delivered if not bounced）  
- [ ] 第三人主張之保證與賠償（Chain-of-title representation + indemnity）  
- [ ] 繼承人/受讓人拘束（Heirs, successors, assigns bound）  
- [ ] 準據法與管轄：台灣法 + 台北地方法院（Taiwan law + Taipei District Court）  
- [ ] 中英版本解釋優先規則（Language precedence clause）

## B. 快速風險閘門 Quick Risk Gates

任一題為「否」即不建議簽署：  
If any answer is “No”, do not execute before revision.

1. 是否存在任何「對客承諾 > 人才授權權利」缺口？  
   Is any customer promise broader than talent-side rights obtained?

2. 是否存在可被主張之人格權、追加報酬、下架請求空間？  
   Is there any opening for moral-rights assertion, extra payment claim, or takedown demand?

3. 刪除機制是否已明確「法令例外 + 退費補償 + 不溯及既往 + Archive 保留」？  
   Does deletion logic clearly include legal carve-out + reimbursement + non-retroactivity + archival retention?

4. 是否明確允許乙方保留自然聲音工作自由，但不得外流本案資料/模型？  
   Is natural-voice livelihood freedom preserved while blocking leakage of project materials/models?

## C. 中英法律術語對照 Legal Term Glossary

| 中文術語 | English Term | 建議定義/使用方式 |
|---|---|---|
| 著作人格權 | Moral Rights | 在台灣脈絡採「不主張」而非「讓與」。 |
| 不主張 | Covenant Not to Assert | 乙方對甲方及其被授權使用人不行使相關人格權。 |
| 權利讓與 | Rights Assignment | 將可讓與之財產性權利移轉予甲方。 |
| 買斷 | Buyout | 一次性對價換取完整且長期權利控制。 |
| 可再授權 | Sublicense | 甲方可授權下游客戶/代理商使用。 |
| 可轉讓 | Transferable / Assignable | 權利可再移轉予第三方。 |
| 衍生成果 | Derivatives / Outputs | 由資料訓練、推理、編輯而生之一切成果。 |
| 歸檔狀態 | Archive Status | 停止新商業生成，永久保留作驗證/法務/合規。 |
| 分離同意/單獨同意 | Separate Consent | 法令要求時對特定處理行為再取得同意。 |
| 視為同意 | Deemed Consent | 合法前提下，逾期未異議即視為同意。 |
| 不溯及既往 | Non-Retroactive Effect | 不影響既有授權、既有交付、既有客戶權利。 |
| 授權鏈 | Chain of Title / License Chain | 從人才到平台再到客戶之權利連續性。 |
| 免權利金 | Royalty-Free | 客戶不需對人才支付持續性權利金或殘值。 |
| 連帶賠償/補償 | Indemnification | 違約或侵權主張時之損害填補與程序費用承擔。 |
| 電子送達 | Electronic Service | 依約定 Email 寄達即生通知效力。 |

## D. 與對客頁面一致性提醒 Consistency Reminders

1. `Broadcast / Global / Buyout` 的命名需與證書和定價頁一致。  
2. 若對客文案寫「100% royalty-free」或「moral rights waived」，人才合約必須有同等或更強條款。  
3. 若對客文案寫「Archive Status permanent retention」，人才合約必須有同等留存授權與用途限縮。  
4. 若對客證書提供 `Voice Affidavit`，人才端必須有固定文案、檔名規則、用途授權。
