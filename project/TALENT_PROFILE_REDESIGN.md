# 配音員後台 / 檔案 重新設計 — 規格

Wing 2026-06-22 design review。Spotify 風 mockup 已核准。整理所有需求 + 分階段。

## 願景
配音員檔案 = Spotify 藝人頁的質感:大頭照 + 關鍵資訊在上,demo 像歌單分類播放。
**不是報名表。** 後台 = 配音員自助管理;前台 = 客戶瀏覽/過濾/找對的人。

## 欄位（完整）
- **大頭照** — 上傳後前端自動裁正方形 + 壓縮(存單一小檔,省雲端)
- **顯示名稱**
- **個人簡介** — 配音員用自己語言打;上線時翻成 繁/簡/英 三版(見「翻譯」)
- **所在地 location** — 國家/地區(關係到時區)
- **可工作時段 / 不可工作時間** — 參考用(例:週末不接、晚上 7 點後不工作)。正式仍會逐案確認,但媒合時方便篩
- **語言** — 每個語言**必須附 demo 才能掛**(沒 demo 不能宣稱會;聽不到無法證明)
- **聲線特質**(可自選):溫暖·親切·沉穩·權威·有活力·低沉·明亮·成熟·年輕·磁性 — 在地化(不直譯)
- **用途/專長**(可自選):廣告·旁白·有聲書·企業形象·教育課程·紀錄片·遊戲角色·動畫·戲劇配音·Podcast·新聞·電話語音(IVR)
- **合作單位 / 經歷** — 跟誰合作過、代表作(可信度,很重要)
- **錄音器材** — 選填(未來 TTS/AI 需要)
- **專業錄音師/錄音室合作** — 有無可配合的專業錄音室(客戶要連線但本人沒設備時可媒合)
- **服務類型(AI/TTS/校對)** — 唯讀顯示 +「想調整請聯絡」(不自己開關,避免動到已簽 AI 合約)
- **真人驗證狀態** — 顯示「✅已通過/⏳待驗證」(驗證仍由 admin 主動發)

## Demo 規則
- **只收 MP3**(不收影片,太佔空間)
- **單檔 ≤ 3 分鐘**(建議 1 分內)、限檔案大小
- **按類型分組**:每類最多 **2 個**;**遊戲角色不限**(每個自己命名角色:慈祥媽媽/冷酷反派…)
- **品質指引(官方語氣)**:建議純人聲(OS)最清晰;可有音樂襯底,但避免過大聲/破音的配樂
- 類型給選單 + 「其他自填」

## 政策
- **編輯要再審**:配音員改檔/傳新 demo **不直接上線**,進「待審」→ admin 確認品質 OK 才發布;品質差可退回。(避免亂 + 確保品質)
- **上線時翻譯**:審核通過要上線時,自動把簡介(+資料)翻成 繁/簡/英 三版,前台依客戶語言顯示

## 翻譯引擎
- 環境**目前沒有** LLM/翻譯 key(只有 Paddle/Resend/Supabase/TapPay)。
- 選項:① Wing 給 Anthropic/DeepL key → 上線時一鍵自動翻譯 + admin 可改;② 先手動(admin 上線時填英文版)。

## 🔮 AI 語意搜尋(Wing 的點子,Phase 3)
別的平台標籤做超細但沒人用超過 10%。所以:標籤只做通用常見的;**客戶找不到就直接打字描述需求**(「我要溫暖親切的女聲」)→ 系統比對 talents 的 標籤+簡介+demo 標題+經歷 keyword → 找出對應人選,不用一直挑標籤。
- v1:關鍵字搜尋(over 標籤+簡介+demo標題)
- v2:語意/embeddings(真正「描述→媒合」)

## 階段
- **Phase 1（先做）**:Spotify 版型 + 全部欄位 + 大頭照轉檔 + 分類 demo(MP3/限量/遊戲不限/品質指引) + 標籤分組在地化 + 語言綁 demo + location/設備/錄音室/經歷/可工作時段 + 服務類型唯讀 + 真人驗證狀態 + 編輯再審 gate
- **Phase 2**:自動翻譯(等 key)
- **Phase 3**:AI 語意搜尋(前台)
